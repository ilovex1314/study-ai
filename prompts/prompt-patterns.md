# Prompt Patterns

更新时间：2026-06-05

## 1. 需求澄清

```text
你是一个资深软件工程师。请基于下面需求，先不要实现，先输出澄清问题和关键假设。

需求：
{需求}

请按以下结构输出：
1. 已知信息
2. 缺失信息
3. 关键风险
4. 建议优先确认的问题，最多 5 个
```

## 2. 技术方案

```text
你是一个负责交付的技术负责人。请为下面需求设计一个可落地方案。

需求：
{需求}

约束：
- 技术栈：{技术栈}
- 时间：{时间}
- 运行环境：{运行环境}

请输出：
1. 目标和非目标
2. 系统边界
3. 核心模块
4. 数据流
5. AI 能力使用点
6. 风险和回滚策略
7. 最小可交付版本
```

## 3. AI 应用选型

```text
请判断下面业务需求应该优先使用 Chat、RAG、Agent、Workflow、Fine-tuning，还是传统程序。

业务需求：
{业务需求}

请输出：
1. 推荐方案
2. 不推荐的方案及原因
3. 是否需要私有知识
4. 是否需要工具调用
5. 是否需要人工确认
6. 成本、延迟、质量风险
7. 验收指标
```

## 4. 结构化输出

```text
请只输出合法 JSON，不要输出 Markdown，不要解释。

任务：
{任务}

JSON Schema：
{schema}

如果信息不足，请在 missing_fields 中列出缺失字段。
```

## 5. 代码修改前分析

```text
你正在一个已有代码库中工作。请先阅读相关上下文，再提出修改计划，不要直接改代码。

目标：
{目标}

相关文件：
{文件路径}

请输出：
1. 当前实现理解
2. 需要修改的文件
3. 修改步骤
4. 风险
5. 验证命令
```
