# study-ai Interactive Learning Plan

Persona: senior frontend/backend engineer who wants to ship AI products, not just learn AI vocabulary.

Pace: 4 focused days, about 60 minutes per day. This is a compact interactive path extracted from the larger 12-week roadmap. It is not a full replacement for the long roadmap; it is the first usable learning product.

Quality score: 88/100

Repair rounds: 1

## Day Map

| Day | Focus | Production Capability |
| --- | --- | --- |
| Day01 | Model cognition and AI capability boundaries | Judge when LLM output can be trusted and when deterministic system control is required. |
| Day02 | Prompt, RAG, and knowledge grounding | Design prompt contracts and a minimal RAG pipeline with measurable retrieval quality. |
| Day03 | Agent orchestration and workflow control | Design an agent execution loop with tools, state, tracing, human confirmation, and failure recovery. |
| Day04 | AI product delivery and operations | Turn demos into products with model gateway, UX, evals, observability, cost control, and deployment. |

## Sources Used

- OpenAI Retrieval docs: vector stores, semantic search, chunking, embedding, retrieval cost.
- OpenAI Agents SDK docs: agents, tools, handoffs, guardrails, tracing.
- Vercel AI SDK docs: TypeScript AI UI, streaming, tools, provider abstraction.
- LangGraph docs: durable execution, checkpointers, interrupts, human-in-the-loop resume.
- Mastra docs: TypeScript agents, workflows, RAG, memory, evals.
- Dify docs/product material: visual workflow, RAG pipelines, tools, agent strategies.

## Quality Gate Notes

- Every question concept is present in the lesson body.
- Every Day includes architecture or data flow.
- Every Day includes hands-on work, a production example, and a counterexample.
- Academic definitions are kept short; implementation, architecture, and production tradeoffs are the center.
